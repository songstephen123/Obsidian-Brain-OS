# 03 — 日常工作流

> 系统配好之后，你每天怎么用？

---

## 一天的流程

### 早上（07:00 自动）

AI 自动生成每日驾驶舱。

**你只需要**：
1. 打开 Obsidian
2. 读 `01-PERSONAL-OPS/01-DAILY-BRIEFS/daily-briefing.md`
3. 读 `03-KNOWLEDGE/01-READING/04-DIGESTS/nightly-digest-昨天日期.md`
4. 开始工作

**驾驶舱告诉你**：
- 今天最重要的 3 件事
- 必须推进但不做完的
- 等待反馈需要催办的
- 低能量时可以做的

**知识摘要告诉你**：
- 昨天处理了什么文章
- 有什么新知识值得看
- 对话中发现了什么洞察

### 白天（随手操作）

#### 收到一篇文章

→ 直接发链接到**文章沉淀频道**

```
你：帮我处理这篇文章 https://example.com/article
```

Agent 会：
1. 抓取文章内容
2. 按 article-note 模板创建笔记
3. 放入 `02-WORKING/01-ARTICLE-NOTES/`
4. 今晚 02:00 Pipeline 自动处理

#### 想到一件事要做

→ 发到**事务管理频道**

```
你：帮我记一下，下周三之前要交报告
```

Agent 会：
1. 写入 `00-INBOX/todo-backlog.md`
2. 设好优先级和截止日期
3. 明天早上自动出现在驾驶舱里

#### 需要做决策

→ 发到**事务管理频道**

```
你：A 方案和 B 方案选哪个？A 便宜但慢，B 贵但快
```

Agent 会帮你分析，并把决策记录到 `decision-queue.md`。

### 下午（15:00 自动提醒）

AI 提醒你检查今天 P0 的进度。

你只需要：
- 看看完成了多少
- 有没有需要调整的

### 晚上（20:00 自动提醒）

AI 提醒你回顾今天。

你只需要：
- 简短反馈：哪些做了、哪些没做、明天要注意什么
- 或者什么都不说，让 AI 自动标记为"无更新"

### 凌晨（02:00-05:00 自动）

Nightly Pipeline 静默运行：

```
02:00  文章整合    ← 处理你白天发的文章
03:00  对话挖掘    ← 从你昨天的对话中提取洞察
04:00  知识放大    ← 连接知识，生成推荐
05:00  知识图谱    ← 可视化（可选）
```

你什么都不用做。明天早上看结果。

## 不同场景的工作流

### 场景 1：读完一篇好文章

```
1. 发链接到文章沉淀频道
2. AI 创建 article note（几秒钟）
3. 当晚 02:00 Pipeline 自动提取知识
4. 第二天早上在 digest 中看到摘要
5. 如果感兴趣，点进去读完整的 domain knowledge card
```

**你的工作量：1 条消息。**

### 场景 2：需要规划一周

```
1. 周一早上 05:10 AI 自动生成 weekly plan
2. 你在事务管理频道看 plan
3. 调整不合理的地方（直接告诉 AI）
4. 本周按 plan 执行
5. 每天早上看驾驶舱，跟踪进度
```

**你的工作量：周一早上 5 分钟调整。**

### 场景 3：和 AI 讨论了一个复杂问题

```
1. 你在任意频道和 AI 讨论
2. 当晚 03:00 Pipeline 自动扫描对话
3. 提取有价值的决策、模式、洞察
4. 写入知识库
5. 第二天在 digest 中看到"昨天对话发现了什么"
```

**你的工作量：0。全自动。**

### 场景 4：知识库越来越乱

```
1. 每周一 01:00 AI 自动运行 knowledge-lint
2. 检查：缺失字段、断链、孤立页面、过期内容
3. 生成报告到 `03-KNOWLEDGE/99-SYSTEM/lint-reports/`
4. 你看一下报告，决定要不要清理
```

**你的工作量：每周 5 分钟看报告。**

## 你应该做的和不应该做的

### ✅ 应该做的

- **随手丢东西进 INBOX**：不要想"这个值不值得记"，先丢进去
- **每天早上看驾驶舱**：5 分钟，了解今天要做什么
- **每周看一次知识摘要**：了解你的知识库在增长
- **定期反馈给 AI**：告诉它哪里做得好、哪里做得差
- **每周调整一次配置**：Prompt、Skill、Cron 都可以微调

### ❌ 不应该做的

- **不要手动整理知识库**：让 Pipeline 做
- **不要在 WORKING 层直接读/写**：那是 AI 的工作台
- **不要跳过每日驾驶舱**：它会越来越准，前提是你坚持看
- **不要一次性装太多功能**：从一个 Profile 开始，逐步扩展
- **不要期望第一天就完美**：系统需要和你一起成长

## 量化预期

### 第一周

- 文章处理开始自动化
- 每日驾驶舱可能不太准（AI 还在学你的习惯）
- 知识库开始有内容

### 第一个月

- 驾驶舱越来越准
- 知识库积累了 50-100 张知识卡
- 你开始习惯"看 digest 而不是翻笔记"
- Pipeline 偶尔出错，你知道怎么修了

### 三个月

- 系统成为你日常生活的一部分
- 知识库成为你的"第二大脑"
- 你开始自定义 Skill 和 Prompt
- 新想法：你开始向朋友推荐这套系统
