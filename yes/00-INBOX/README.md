# 📥 00-INBOX — 统一收件口

> **Everything goes here first. No exceptions.**

---

## 用途

这是整个知识系统的**唯一入口**。所有新想法、待办、文章链接、随手记录，先放这里，再做分流。

- 不追求全量堆积，追求**后续可判断、可推进、可清理**
- 不是最终工作台：进入后会被分流到「今日驾驶舱 / 决策队列 / 当前承诺 / 本周推进盘」

---

## 文件说明

| 文件 | 用途 |
|------|------|
| `todo-backlog.md` | 待办总库（SSoT — Single Source of Truth） |
| `archive/` | 已归档待办（按月存放） |

---

## 待办格式

```markdown
- [ ] [P1] 事项名称
  - 领域：工作 / 个人 / 家庭 / 健康 / 财务 / 社交
  - 重要性：高 / 中 / 低
  - 紧急性：高 / 中 / 低
  - 杠杆性：高 / 中 / 低
  - 截止：YYYY-MM-DD 或 无
  - 下一动作：一句话写清下一步
  - 关联项目：项目名（可选）
  - 是否必须本人：是 / 否
  - 是否可委派：是 / 否
  - 状态：待判断 / 待执行 / 等待反馈 / 已排期 / 已完成 / 放弃
```

## 使用原则

1. **先进 INBOX，再分流** — 任何新事项不要直接跳到执行
2. **每日或每周做一次分流** — 把 todo-backlog 里的事项推到对应驾驶舱
3. **不在这里做判断** — INBOX 只做收集，判断在驾驶舱完成
